2.1 WHERE - Filtrare i Dati

Definizione

La clausola WHERE filtra i risultati della query, restituendo solo le righe che soddisfano una condizione specifica. E il principale strumento per selezionare sottoinsiemi di dati.

Finora abbiamo visto come estrarre dati con SELECT. Ma nelle situazioni reali, le tabelle contengono migliaia o milioni di righe, e raramente ci serve vedere tutto. La clausola WHERE ci permette di specificare esattamente quali righe vogliamo.

Analogia pratica

Se SELECT e "mostrami i libri", WHERE e "mostrami solo i libri di fantascienza" o "mostrami solo i libri pubblicati dopo il 2020". Aggiunge una condizione al tuo comando.

Sintassi

SQL - Sintassi WHERE
SELECT colonne
FROM tabella
WHERE condizione;

Esempi base

SQL - Filtrare per valore testuale
-- Mostra solo i dipendenti del dipartimento IT
SELECT * 
FROM dipendenti 
WHERE dipartimento = 'IT';
SQL - Filtrare per valore numerico
-- Mostra solo i prodotti con prezzo superiore a 100
SELECT * 
FROM prodotti 
WHERE prezzo > 100;

2.2 Operatori di confronto

Gli operatori di confronto sono il cuore delle condizioni WHERE. Ti permettono di confrontare valori nelle colonne con valori specifici.

Operatore Significato Esempio SQL Righe restituite
= Uguale a WHERE categoria = 'Elettronica' Solo prodotti di categoria "Elettronica"
<> o != Diverso da WHERE stato <> 'annullato' Tutti gli ordini tranne quelli annullati
> Maggiore di WHERE stipendio > 40000 Dipendenti con stipendio oltre 40000
< Minore di WHERE quantita < 10 Prodotti con meno di 10 pezzi in stock
>= Maggiore o uguale WHERE eta >= 18 Utenti maggiorenni (18 incluso)
<= Minore o uguale WHERE sconto <= 50 Prodotti con sconto fino al 50%
Regola fondamentale

I valori testuali (stringhe) devono essere racchiusi tra apici singoli 'valore'. I numeri NO.

Confronto con date

SQL - Confronto con date
-- Dipendenti assunti dopo il 1 gennaio 2020
SELECT * 
FROM dipendenti 
WHERE data_assunzione > '2020-01-01';

-- Ordini dell'anno 2023
SELECT * 
FROM ordini 
WHERE data_ordine >= '2023-01-01' 
  AND data_ordine <= '2023-12-31';

2.3 AND / OR - Combinare condizioni

Spesso una singola condizione non basta. Gli operatori logici AND e OR permettono di combinare piu condizioni nella stessa clausola WHERE.

AND

Restituisce la riga solo se TUTTE le condizioni sono vere. Basta che una sia falsa perche la riga venga esclusa.

SQL - Operatore AND
-- Dipendenti del dipartimento IT con stipendio superiore a 40000
SELECT * 
FROM dipendenti 
WHERE dipartimento = 'IT' AND stipendio > 40000;

OR

Restituisce la riga se ALMENO UNA delle condizioni e vera. Basta che una sia vera perche la riga venga inclusa.

SQL - Operatore OR
-- Dipendenti del dipartimento IT oppure del Marketing
SELECT * 
FROM dipendenti 
WHERE dipartimento = 'IT' OR dipartimento = 'Marketing';
Attenzione: priorita degli operatori

AND ha priorita su OR! Senza parentesi, il risultato potrebbe essere diverso da quello che ti aspetti. Usa sempre le parentesi quando combini AND e OR.

SQL - Combinare AND e OR con parentesi
-- Dipendenti IT con stipendio > 40000 OPPURE qualsiasi Marketing
SELECT * 
FROM dipendenti 
WHERE (dipartimento = 'IT' AND stipendio > 40000) 
   OR dipartimento = 'Marketing';

2.4 IN - Verificare appartenenza a un elenco

Definizione

L'operatore IN verifica se un valore appartiene a un elenco predefinito. E una sintassi piu pulita e leggibile rispetto a molteplici condizioni OR.

SQL - Con OR (verboso)
SELECT * FROM dipendenti 
WHERE dipartimento = 'IT' 
   OR dipartimento = 'Marketing' 
   OR dipartimento = 'Vendite';
SQL - Con IN (elegante)
SELECT * FROM dipendenti 
WHERE dipartimento IN ('IT', 'Marketing', 'Vendite');
SQL - NOT IN per escludere
-- Tutti i dipendenti TRANNE quelli di HR e Amministrazione
SELECT * FROM dipendenti 
WHERE dipartimento NOT IN ('HR', 'Amministrazione');

2.5 IS NULL - Gestire i valori mancanti

Cos'e NULL?

NULL in SQL rappresenta l'assenza di un valore. Non e zero, non e una stringa vuota, non e la parola "null" - e proprio "nessun dato presente".

NULL non e... Esempio Differenza
Zero (0) stipendio = 0 Zero e un valore numerico. NULL significa "non sappiamo lo stipendio"
Stringa vuota ('') email = '' Stringa vuota e un valore stringa. NULL significa "email non inserita"
Errore comune

NULL non puo essere confrontato con =. La query WHERE bonus = NULL non funziona! Usa sempre IS NULL o IS NOT NULL.

SQL - Gestione NULL
-- SBAGLIATO: non funziona!
SELECT * FROM dipendenti WHERE bonus = NULL;

-- CORRETTO: trovare righe CON valori mancanti
SELECT * FROM dipendenti WHERE bonus IS NULL;

-- CORRETTO: trovare righe SENZA valori mancanti
SELECT * FROM dipendenti WHERE bonus IS NOT NULL;

2.6 ORDER BY - Ordinare i risultati

Definizione

La clausola ORDER BY ordina i risultati della query in base ai valori di una o piu colonne. Puoi scegliere l'ordine crescente (ASC) o decrescente (DESC).

Direzione Significato Numeri Testo Date
ASC (default) Crescente (Ascending) 1, 2, 3... A, B, C... Piu vecchio prima
DESC Decrescente (Descending) ...3, 2, 1 ...C, B, A Piu recente prima
SQL - Ordinamento
-- Dipendenti in ordine alfabetico per cognome
SELECT * FROM dipendenti ORDER BY cognome ASC;

-- Prodotti dal piu caro al meno caro
SELECT nome_prodotto, prezzo 
FROM prodotti 
ORDER BY prezzo DESC;

-- Ordinamento multiplo: prima per dipartimento, poi per stipendio
SELECT * FROM dipendenti 
ORDER BY dipartimento ASC, stipendio DESC;

Combinazione completa: WHERE + ORDER BY + LIMIT

SQL - Query completa
-- Top 5 prodotti piu costosi della categoria Elettronica
SELECT nome_prodotto, prezzo 
FROM prodotti 
WHERE categoria = 'Elettronica'
ORDER BY prezzo DESC
LIMIT 5;

Ordine corretto delle clausole SQL

SELECT   colonne
FROM     tabella
WHERE    condizione
ORDER BY colonna [ASC|DESC]
LIMIT    n;

Ricorda: questo ordine e obbligatorio!
